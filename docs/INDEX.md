## 🚀 Quick Start

### Installation

```bash
cd ~/Desktop/open_clock_pit

# Create & activate virtual environment
python3 -m venv venv
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

### Running OpenCockpit

```bash
# Standard run
python OpenCockpit/main.py

# With custom serial port
export MSP_PORT=/dev/ttyACM0
python OpenCockpit/main.py

# With debug output
python OpenCockpit/main.py 2>&1 | tee opencockpit.log
```

### Web UI

Once running, access the web UI:
```
http://<rpi-ip>:5000
```

## 🔧 Configuration Files

### config.json (Runtime Configuration)

Auto-generated by web UI, or manually create:

```json
{
  "displays": {
    "Display_1": "HUD_0.85",
    "Display_2": "MAP_0.96",
    "Display_3": "MFD_0.96",
    "Display_4": "INFO_0.96"
  },
  "hardware": {
    "gpio_pins": {
      "Display_1": {"cs": 5, "dc": 25, "rst": 22},
      "Display_2": {"cs": 6, "dc": 24, "rst": 27},
      "Display_3": {"cs": 13, "dc": 23, "rst": 17},
      "Display_4": {"cs": 19, "dc": 26, "rst": 16}
    }
  },
  "fc": {
    "port": "/dev/serial0",
    "baudrate": 115200,
    "protocol": "auto"
  }
}
```

### Environment Variables

```bash
# Flight Controller connection
export MSP_PORT=/dev/ttyACM0
export MSP_BAUD=115200
export MSP_PROTOCOL=msp

# Reconnection behavior
export MSP_RECONNECT_IDLE_S=3.0
export MSP_RECONNECT_BACKOFF_S=0.5
```

## ⚠️ Common Issues

### Display Not Initializing

1. Check GPIO pins in config.json
2. Verify SPI bus not in use by other process
3. Check power supply (should be 3.3V stable)
4. Look for error messages in terminal

### No Telemetry Appearing

1. Check serial port: `ls -la /dev/ttyACM0` or `/dev/serial0`
2. Verify baud rate (typically 115200)
3. Test with: `screen /dev/ttyACM0 115200`
4. Check FC is outputting MSP protocol (INAV/ArduPilot)

### Flickering Displays

1. Reduce framerate in config.json
2. Check SPI baudrate (try 26MHz instead of 52MHz)
3. Look for lock contention in logs

## 🛠️ Development

### Adding a New Display Module

1. Create `NEW_MODULE_pi_NNN.py` (copy existing module as template)
2. Implement required render functions:
   - `render_hud()` or `render_map()` or `render_mfd_dynamic()` etc.
3. Define `DISPLAY_CONFIG` dict with driver, size, rotation
4. Add to `MODULE_MAP` in main.py
5. Assign GPIO pins in DISPLAY_HARDWARE_MAP
6. Select in SELECTED_DISPLAYS or web UI

### Adding a New Data Field

1. Add MSP request to `MSP_Read_pi.py`
2. Create parse function for response
3. Add to shared `data` dict
4. Use in display modules: `snap["field_name"]`
5. Expose in web API `/api/telemetry`

## 🐛 Debugging Tips

### Enable Verbose Logging

```python
# In main.py or MSP_Read_pi.py
import logging
logging.basicConfig(level=logging.DEBUG)
```

### Test MSP Connection

```python
import serial
ser = serial.Serial('/dev/serial0', 115200)
ser.write(b'$M<\x00\x6c\x68')  # MSP_ATTITUDE request
response = ser.read(50)
print(response.hex())
```

### Profile Rendering

```python
import time

# In display_loop
t0 = time.time()
module.render_hud(...)
t_render = time.time() - t0

t0 = time.time()
disp._block(0, 0, w-1, h-1, buf)
t_spi = time.time() - t0

print(f"Render: {t_render*1000:.1f}ms, SPI: {t_spi*1000:.1f}ms")
```
